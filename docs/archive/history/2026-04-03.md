## 摘要

- 落地 A.1/A.2：移除仓库内已跟踪平台二进制（`styio*`），构建产物统一输出到 `build/bin`，测试目标路径改为 `$<TARGET_FILE:styio>`。
- 落地 A.3（最小 CI）：新增 `.github/workflows/ci.yml`，包含 configure/build/security/pipeline/milestone smoke。
- 落地 B.1/B.2/B.3：
  - 新增 `StyioLexError` / `StyioTypeError`。
  - `Tokenizer` 修复未闭合字符串、未闭合块注释、行注释 EOF 边界。
  - `main` 引入分阶段错误分类与稳定退出码，新增 `--error-format=text|jsonl`。
- 落地 B.4/B.5（首版）：
  - 新增 `tests/fuzz/`（`styio_fuzz_lexer`、`styio_fuzz_parser`）与 seed corpus。
  - CI 加入 fuzz smoke（`fuzz_smoke` 标签）。
  - 新增夜间深跑工作流 `.github/workflows/nightly-fuzz.yml` 并归档 fuzz artifacts。
- 落地 C.1/C.2（首段）：
  - 新增 `CompilationSession`，由会话集中接管 Token/Context/AST/IR 生命周期。
  - 新增 `StyioHandleTable` 空壳类型。
  - `StyioContext` 补析构释放 `ast_repr`。
- Unicode 边界收敛：
  - 新增 `StyioUnicode` 防腐层（`src/StyioUnicode/Unicode.hpp/.cpp`）。
  - Parser/Tokenizer 不再直接调用 `std::is*`，统一走 `StyioUnicode::*`。
  - `STYIO_USE_ICU=ON` 时仅在防腐层内部启用 ICU codepoint 分类能力。
- 里程碑回归修复（M1–M8 全绿）：
  - 修复 Tokenizer 空白字符循环控制：空白 token 产生后立即 `continue`，避免缩进空白被误记为 `UNKNOWN`。
  - 补充 runtime 读取路径兼容：`tests/mX/...` 自动映射到 `tests/milestones/mX/...`（文件存在时）。
  - 增加 runtime 错误旗标并在 `main` 中映射为稳定非零退出码（修复 `m5_t06_fail_fast`）。
  - 修复 mixed-match 字符串提升的生命周期错误（不再返回栈内临时缓冲区指针）。
- Runtime C ABI 首轮收敛：
  - `styio_file_open*` 从 `exit(1)` 改为返回 `0` 句柄。
  - 句柄从裸指针整数改为受控表映射。
  - 新增 `styio_free_cstr` 并在写文件追加换行路径回收拼接内存。
