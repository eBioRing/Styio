## Checkpoint（M9/M10 Contract Audit：StdIO 冻结契约与实际实现对齐）

### 当前状态

- 目标：
  - 审核另一批标准流改动后的真实契约，并把 M9/M10 milestone、five-layer、SSOT 引用和 workflow 文档重新对齐。
- 发现的问题：
  - `M9` 把 `$"..."` format string 当作 stdout 里程碑的冻结验收，但当前 format string 仍不是这批标准流工作的可靠完成条件。
  - `M10` 文档和 fixture 把 `(<< @stdin)` 写成“字符串原样回显”，但真实实现和 `p14_stdin_pull` five-layer golden 都已经冻结成 `styio_cstr_to_i64()` 标量路径。
  - `styio_stdin_read_line()` 在 milestone task 文本里被误写成 malloc/显式 free 契约，实际实现是 thread-local borrowed buffer。
- 修复内容：
  - `tests/milestones/m9/`：
    - 将 `t06_stdout_fmtstr` 正式收口为 `t06_stdout_concat`，验收内容改为 `"x=" + x -> @stdout`，不再把 format string 作为 M9 阻塞项。
  - `tests/milestones/m10/`：
    - `t02_stdin_pull` / `t04_stdin_multi_pull` fixture 与 expected 改为 numeric stdin 数据，和当前 `p14_stdin_pull` 契约保持一致。
  - `docs/milestones/2026-04-08/`：
    - 更新 M9/M10 验收文字、Task 说明、Index 总览，显式写明：
      - 无 wrapper definitions
      - canonical `-> @stdout/@stderr`
      - accepted shorthand `>> @stdout/@stderr`
      - `(<< @stdin)` 当前冻结为 numeric instant-pull contract
      - `styio_stdin_read_line()` 为 borrowed buffer
  - `docs/plans/Standard-Streams-Plan.md`、`docs/design/Styio-Language-Design.md`、`docs/design/Styio-Symbol-Reference.md`、`docs/assets/workflow/FIVE-LAYER-PIPELINE.md`、`docs/assets/workflow/TEST-CATALOG.md` 同步收口。
  - 新增 ADR：`docs/adr/ADR-0117-stdio-milestone-contract-audit.md`。

### 下一步

1. 若要推进更直觉的 string-based `(<< @stdin)`，必须单独开一个 checkpoint，连同 `p14_stdin_pull` 和相关 goldens 一起迁。
2. format string 继续按独立特性轨道收口，不再借道 M9 标准流验收。
3. 在完成这次契约收口后，再决定是否需要补额外的 parser/runtime regression。

### 风险

1. 这次 checkpoint 明确选择“文档/fixture 对齐实现”，而不是修改 `(<< @stdin)` 行为；如果后续决定切字符串语义，将引起 one-shot golden 变更。
2. `../design/Styio-Language-Design.md` 现在必须同时描述 line-iterator 的字符串路径和 instant pull 的 numeric path，后续 typed string 化时必须记得同步两处。
