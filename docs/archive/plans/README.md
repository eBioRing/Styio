# Archived Plans

**Purpose:** Define the scope of absorbed or superseded implementation plans under `docs/archive/plans/`; these files preserve sequencing history after their durable rules have been moved into active docs.

**Last updated:** 2026-04-16

## Scope

1. Store completed, absorbed, or superseded implementation plans.
2. Preserve rollout rationale, rollback notes, and historical sequencing without treating these files as active SSOT.
3. Keep still-open implementation plans in `../../plans/`.

## Rules

1. Move a plan here only after its living rules have been lifted into design/specs/runbooks/workflow docs.
2. Active docs should not rely on files here as default reading order.
3. Historical detail is allowed to remain in archived plan bodies; active docs should link here only for provenance.
