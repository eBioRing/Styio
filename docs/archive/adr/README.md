# Archived ADRs

**Purpose:** Define the scope of absorbed architecture decision records under `docs/archive/adr/`; these files preserve decision provenance after the active rule has been lifted into owning SSOT documents.

**Last updated:** 2026-04-16

## Scope

1. Store ADRs whose durable conclusions have already been absorbed into active docs.
2. Preserve context, alternatives, and consequences for audit and historical reasoning.
3. Keep only not-yet-absorbed decision records in `../../adr/`.

## Rules

1. Do not use archived ADRs as default maintenance input.
2. Before archiving an ADR, ensure the surviving rule is present in design/specs/runbooks/workflow docs.
3. Use archived ADRs only when the project needs to answer why a rule was chosen, not what the current rule is.
