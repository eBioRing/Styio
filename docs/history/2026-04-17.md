## Checkpoint（Docs / Ecosystem：cross-repo sync risk intake）

### 当前状态

- 已对本轮三仓同步做了一次 review-style 风险扫描，并将属于 `styio-nightly` 自身维护面的条目接回 active docs。
- 本仓当前没有新增的 compiler-side 主线功能回归被归责到 `nightly`；主要记录的是本仓 repo hygiene / fixture 维护面上的低风险问题：
  1. 根 `.gitignore` 现在统一忽略 `tmp/`、`build-*`、`*.tmp`、`*.log` 和常见 cache/env 目录。
  2. 这有利于本地清理和三仓统一 hygiene，但也意味着未来若有人想把 repro fixture、恢复样本或手工诊断资产放在 temp/build 风格路径下，可能会被默认忽略。
- 该问题已接入 [../rollups/NEXT-STAGE-GAP-LEDGER.md](../rollups/NEXT-STAGE-GAP-LEDGER.md)，作为低风险但需要持续注意的 active ledger 条目。

### 下一步

1. 后续若要提交新的 repro fixture、恢复样本或诊断工件，优先避免使用 `tmp/`、`build-*`、`*.tmp`、`*.log` 风格路径。
2. 若确实需要沿用这些路径名，必须同批补显式 negate rule，并让 Docs / Ecosystem 与 Test Quality 一起 review。
3. 保持 `nightly` active gap ledger 只记录本仓责任边界内的问题；`spio` 和 `view` 的执行层缺陷继续分别记到各自仓库。

### 复现命令

```bash
git check-ignore -v tmp/sample.txt build-debug/probe.txt sample.tmp sample.log
```

### 风险

1. 这是维护与交付可见性风险，不是当前语言/编译器语义回归。
2. 若后续团队只记得“统一 ignore”而忘了“显式反忽略 tracked fixture”，问题会在代码 review 阶段之前被隐藏。

## Checkpoint（Docs / Ecosystem：FG3 shared ignore baseline）

### 当前状态

- `nightly` 已把 shared file-governance baseline 写回根 `.gitignore`，并冻结 `docs/**`、`tests/**` 的 temp/build 风格 fixture negate 规则。
- `scripts/repo-hygiene-gate.py` 现在不只看垃圾产物和大文件，也会强校验 required `.gitignore` patterns 与关键治理文档接线。
- 这轮的目标不是扩大忽略范围，而是把“允许忽略什么、哪些 tracked fixture 必须显式反忽略”做成脚本级门禁。

### 下一步

1. 若未来要在 `benchmark/`、`examples/` 或其它新根下追踪 temp/build 风格 repro 资产，必须先扩展显式 negate rules，再扩 gate。
2. 继续把这套 baseline 与 `styio-spio`、`styio-view` 的 mirror/gate 保持同级，不允许 nightly 单仓继续漂移。

### 复现命令

```bash
python3 scripts/repo-hygiene-gate.py --mode tracked
python3 scripts/docs-audit.py
```

### 风险

1. `docs/**` 和 `tests/**` 之外的新 tracked repro 根仍然需要显式建模，否则 broad ignore 仍会隐藏它们。
2. 这轮只冻结 shared baseline，不替代 team 级 fixture 命名审查。
